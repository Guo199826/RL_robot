Part A

倒立摆 Swing-up：
不确定性下的「经典控制 vs 强化学习」

一句话主旨：模型准时，model-based 最优；模型不准时，带域随机化的 RL 才换来鲁棒性 —— 而在连续控制上，off-policy 的 SAC 比 PPO 样本效率高得多。

Part A · Slide 1

问题设定 & 为什么这是个「诚实」的对照任务

  • 任务：力矩受限的倒立摆 swing-up（θ=0 为竖直向上的不稳定平衡）。最大力矩不足以直接举起 → 必须先注入能量摆上去、再在顶端平衡。任务本质是欠驱动（underactuated）。
  • 为什么不用 reach/push 机械臂：在全驱动、反馈友好的系统上，调好的反馈控制器对质量/摩擦误差天生鲁棒 → 在那里说「RL 打赢经典控制」不诚实。
  • 在欠驱动任务上，单纯反馈律理论上就到不了目标 —— 要么需要准确模型（能量整形），要么需要学习。这才是干净的对比场景。

核心问题

在参数不确定（质量 / 长度 / 阻尼未知）下，哪种方法扛得住？

Part A · Slide 2

方法：动力学 · 奖励 · 域随机化 · 五个控制器

动力学（均匀杆）& 奖励

I=m·l²/3，重力力矩 m·g·(l/2)·sinθ，阻尼 b·θ̇，dt=0.05，|u|≤2，|θ̇|≤8；名义 m=1,l=1,g=10,b=0。

reward = −( θ_norm²  +  0.1·θ̇²  +  0.001·u² )
  • θ_norm²（主导）：离目标角度的代价，驱动摆到并停在顶端（max≈9.87）。
  • 0.1·θ̇²：罚角速度，鼓励到顶静止稳住。
  • 0.001·u²：力矩正则，权重极小、只在等效解里偏好省力。
  • 为何 dense 而非 sparse：欠驱动下稀疏奖励几乎学不动；稠密奖励给出连续的能量整形梯度。

量纲排序：位置 ≫ 速度 ≫ 力矩。所有 RL 共用同一套奖励；经典控制器不用奖励（靠模型）。

域随机化（每个 episode reset 时重采样）

参数方式范围
质量 m乘性×[0.6, 1.4]（±40%）
长度 l乘性×[0.7, 1.3]（±30%）
阻尼 b加性[0.0, 0.15]

m,l 一变，惯量 I 与重力力矩都变 → swing-up 能量与顶端线性化都偏离名义。RL 每个 episode 见到不同的摆 → 学到跨整个参数分布都能用的策略（sim2real 核心）。经典 Energy+LQR 始终基于固定名义模型，参数一随机就必然失配。

五个控制器

  • PID（朴素反馈） —— −(Kp·θ+Kd·θ̇)，预期失败的基线。
  • Energy+LQR（model-based） —— 能量泵 + 顶端 LQR 接住。
  • PPO / SAC —— 稠密奖励 + 域随机化训练。
  • Model-based + RL 残差 —— SAC 在 Energy+LQR 之上学修正量。

SAC vs PPO：样本效率

平均评测回报 vs 训练步数（两者均带域随机化训练）
结论SAC ≈ 12k 步即达近最优 swing-up（return≈−310）；PPO 需 150–200k 步（≈15× 样本，200k 才约 −350）。
  • 差距来自 off-policy 的 replay 复用 + 熵正则探索 vs on-policy 的样本利用率。
  • 有限样本预算下做连续控制，SAC 的样本效率 ≫ PPO —— 这就是「SAC 更适合连续控制」的具体证据。

该看的收敛信号：ent_coef 从 0.40 自适应衰减到 ~0.072（探索→利用）+ eval return 上升。

总结论

  1. 经典 model-based 名义最棒，不确定性下很脆。
  2. RL（域随机化）牺牲一点名义最优性，换来巨大鲁棒性。
  3. 残差混合：保留 model-based 能力 + 叠加 RL 鲁棒性，两者兼得。
  4. SAC > PPO：样本高效的连续控制。
讲解点一个微缩的 sim2real 故事 —— 域随机化是从「我的模型里能用」到「现实有偏差也能用」的桥梁。

SAC 的「稳态力矩抖动」只在随机部署才出现

SAC vs PPO 力矩-时间：确定性（取均值）vs 随机（按分布采样）
先猜错→数据纠正直觉以为 SAC 因最大熵会在顶端抖，但确定性策略并不抖，抖动只在按分布采样时出现。
  • 确定性（GIF 用的方式）：SAC/PPO 都平滑收敛到恒定保持力矩 ≈0.19；SAC std≈0.000。
  • 随机：SAC 刻意保留高熵（ent_coef≈0.072）→ 力矩剧烈抖动（std≈0.31）；PPO 熵≈0 → 不抖。
  • 机制：SAC 最大熵目标 E[Σr+α·H(π)] + 顶端 Q 在动作维近平 + u² 惩罚极小 → 采样方差保留到稳态。
部署建议真机部署取均值（deterministic）即平滑；若需带探索在线运行，再加 Δu² 平滑惩罚/减小目标熵。这是连续控制「探索 vs 平滑」权衡的干净例证。
Part B

用 RL 在线整定 PID 增益
（FetchReach 机械臂）

一句话主旨：不要让 RL 取代 PID，而是让 RL 去「调」PID。让 SAC 实时输出 [Kp,Ki,Kd]、由内层 PID 闭环驱动机械臂 —— 平均增益≈人工精调，但能沿轨迹动态调节：过冲↓3×、调节时间↓6×，稳态持平。

Part B · Slide 1

问题设定 & 为什么这样设计

  • 任务：FetchReach-v4（7-DoF 机械臂，末端到达 3D 目标），dense 奖励，全驱动、反馈友好。
  • 核心想法：不让网络直接吐速度，而是每一步输出一组 PID 增益 [Kp,Ki,Kd]，再由内层 PID 用 误差=目标−末端位置 算速度去驱动。

为什么这样做（卖点）

  1. 结构可解释 / 可上线：底层始终是工程师信得过的 PID，RL 只负责「拧旋钮」。
  2. 天然增益调度：固定 PID 只能用一组增益走全程；RL 按状态调 = 自动学到「远处大增益快逼近、近处小增益抑过冲」。
  3. 诚实对照：固定 PID 是强基线，干净回答「自适应到底带来了什么」。
讲解点「learning + control」混合范式 —— 保留经典控制的结构与安全性，用学习补上自适应能力。
Part B · Slide 2

方法：动作重定义 · 内层 PID · 奖励 · 训练

动作空间重定义（核心）

Kp = (a₀+1) × 2.5  → [0, 5]
Ki = (a₁+1) × 0.5  → [0, 1]
Kd = (a₂+1) × 1.0  → [0, 2]

内层 PID 闭环（每个 env step 内）

error      = desired_goal − gripper_pos
integral  += error; clip(integral,−1,1)   # 抗饱和
derivative = error − prev_error
v          = Kp·error + Ki·integral + Kd·derivative
action[:3] = clip(v, −1, 1)               # 速度指令

奖励整形（快、准、稳）

reward = −dist
       − 0.1·speed²   (dist<3cm：靠近罚速度→抑过冲)
       + 0.02         (dist<1.5cm：停留奖励→稳态)

训练配置

  • 算法 SAC（MultiInputPolicy，off-policy），lr=1e-3，50k 步。
  • 训练加观测噪声（obs 0.005 / goal 0.002）—— 见 Slide 5 诚实声明。
关键经典 PID 增益是人调死的；RL-PID 把「调增益」交给学习，且逐步调。
关于量纲调节每个塑形项的权重,要让它的数值大小"恰好在它该起作用的区域"与主项(-dist)同量级,在别处可忽略
Part B · Slide 2 展开 · Reward 尺度设计

那两个权重 0.1 / 0.02 是怎么调的

唯一原则每个塑形项的权重，要让它的数值大小「恰好在它该起作用的区域」与主项 −dist 同量级，在别处可忽略。门控定范围，权重定强度，函数形状定选择性。

主项定「标尺」

一切以 −dist（米）为基准，它在各区域的量级决定塑形项要多大才「被听见」：

区域dist|−dist| 量级
起点≈0.145 mO(0.1)
靠近区<0.03 mO(0.01–0.03)
稳态≈0.0015 mO(0.001)

① 速度惩罚 0.1·speed²（dist<3cm）

让「近处高速」≈「离目标 3cm」一样贵 → 交叉点落在会过冲的高速档，慢速精修近乎免罚：

speed0.1·speed²vs 主项(≈0.03)
0.5（快·要过冲）0.025同量级 → 强罚
0.2（中）0.004≈1/7 温和
0.05（在收敛）0.00025可忽略

平方 → 只精准打击大速度；空间门控在 3cm 内 → 远处仍可高速冲（编码了「远冲近稳」的增益调度）。

② 停留奖励 +0.02（dist<1.5cm）

制造一个「粘性势阱」压稳态误差。关键约束：bonus 必须压过半径处的距离惩罚 → 0.02 > 0.015，净奖励才在圈内翻正。

位置净 reward
边缘 dist=0.015−0.015 + 0.02 = +0.005（转正）
圆心 dist→0+0.02
刚出界 dist=0.016−0.016（无 bonus）

边界处出现 ≈0.021 的断崖：圈内严格优于「刚出圈」→ 阱才粘得住。若设成 <0.015，半径处仍为负，势阱不粘。

调法（有方向的手调，不是瞎试）

  1. 先用「局部同量级 / 翻正条件」锁定数量级（0.1 而非 1 或 0.01；0.02 而非 0.2）。
  2. 再小范围微调，看 pid_vs_rl_pid_comparison.png 的过冲峰值 + 调节时间是否改善。

Part B · Slide 5 ·「灵魂」

核心结论 & 诚实声明

核心结论

  1. RL 学到的平均增益 ≈ 人工精调好 PID（3.39/0.28/1.00 ≈ 3.0/0.3/1.0）→ 解物理合理 + 给了强基线。
  2. 稳态持平、瞬态完胜（过冲↓3×、调节↓6×）。差距来源不是「更好的增益」，而是「会变的增益」 —— 沿轨迹做固定增益做不到的自适应增益调度。
  3. 范式价值：RL 作为在线自整定器叠加在 PID 之上，既保留经典控制的结构/可解释/安全，又拿到自适应的瞬态性能。
总讲解点一个微缩的「learning-augmented control」案例 —— 用学习去自适应整定一个可信赖的经典控制器，不牺牲稳态与可解释性地显著改善瞬态。

Wrap-up

一页总结：什么时候用什么

Part A倒立摆

  • 模型准 → 经典 model-based 最优且便宜。
  • 模型不准 → 带域随机化的 RL 鲁棒，SAC 1.00 vs LQR 0.27。
  • 残差混合两者兼得；SAC ≫ PPO 样本效率（12k vs 200k）。

Part BRL 整定 PID

  • 让 RL 调 PID 而非取代 → 可解释、可上线、安全。
  • 平均增益≈人工精调；价值在自适应增益调度：过冲↓3×、整定↓6×。
贯穿两个项目的一句话RL 的正确位置不是「取代经典控制」，而是「在模型不确定 / 需要自适应的地方，补上经典控制做不到的鲁棒性与调度」—— 域随机化与 learning-augmented control 是这座桥。
